## 1.OpenCV 를 이용하여 개발된 제품을 조사하고 기능을 설명하라

OpenCV는 실시간 이미지 프로세싱에 최적화되어 있어 보안, 자동차, 의료 등 다양한 산업 분야에서 핵심 기술로 사용된다.

**대표 기능 :**

* 자율주행 자동차의 차선 이탈 방지 기능 : 도로의 차선을 실시간으로 감지하여 차량이 경로를 벗어나지 않도록 조향을 보조한다.
* 객체 인식 기능 : 주변 차량, 보행자, 신호등 및 표지판을 인식하여 충돌을 방지하고 주행 전략을 세운다.


## 2.소프트웨어 분야의 용어인 라이브러리의 의미를 자세히 설명하라

소프트웨어 분야의 용어 라이브러리의 의미는 특정 기능을 수행하는 함수나 클래스들을 미리 작성하여 모아놓은 코드 집합체들을 말한다.

**핵심 기능 :**

* 재사용성: 동일한 기능을 매번 구현하지 않고 이미 검증된 코드를 가져다 쓸 수 있다.
* 효율성: 개발 시간을 단축하고 전체 코드의 길이를 줄여 관리, 개발 시간의 효율을 높일 수 있다.
* 모듈화: 독립된 기능을 분리하여 관리하므로 유지보수가 용이해진다.


## 3.오픈소스의 의미를 설명하라

오픈소스는 소프트웨어의 저작권자가 소스 코드를 공개하여 누구나 자유롭게 확인, 수정, 배포할 수 있도록 허용한 소프트웨어이다.

**핵심 특징 :**
	
* 투명성: 소스 코드가 공개되어 있어 프로그램의 작동 원리를 누구나 확인할 수 있다.
* 자유로운 사용: 저작권자가 정한 라이선스 규정 내에서 누구나 무료로 사용하거나 재배포 가능하다.
* 집단 지성: 전 세계 개발자들이 오류를 수정하고 기능을 개선하여 소프트웨어의 완성도를 높인다.


## 4.Windows 운영체제의 환경변수 중 Path 변수의 기능을 설명하라

Windows 환경변수 Path는 실행 파일을 찾을 때 시스템이 탐색하는 디렉터리 경로의 목록을 저장한 변수이다.

**핵심 기능 :**

* 명령어 실행 간소화: 실행 파일이 있는 전체 경로를 입력하지 않고 파일 이름만 입력해도 프로그램을 실행할 수 있게 된다.
* 자동 탐색: 터미널이나 실행 창에서 명령어를 입력하면, 운영체제는 현재 디렉터리에 해당 파일이 없을 경우 Path에 등록된 경로들을 순서대로 뒤져서 파일을 찾는다.
* 소프트웨어 연동: OpenCV나 Python 같은 도구들이 서로의 실행 파일이나 라이브러리를 참조할 때 Path를 통해 위치를 확인한다.


## 5.프로그램 개발과정인 편집 컴파일 링크 실행 단계를 각각 설명하라

	## 특히 링크단계에서 라이브러리의 역할에 대하여 설명하라

* 편집 : 사용자가 프로그래밍 언어를 사용하여 소스 코드를 작성하고 저장하는 단계이다.
* 컴파일 : 소스 코드를 컴퓨터가 이해할 수 있는 기계어 형태인 오브젝트 파일로 변환하는 과정이다. 이때 코드의 문법 오류를 검사한다.
* 링크 : 여러 개의 오브젝트 파일과 프로그램에서 호출한 라이브러리 파일들을 결합하여 하나의 완성된 실행 파일을 만드는 단계다.
* 실행 : 생성된 실행 파일을 메모리에 적재하여 프로세스로 동작시키는 최종 단계다.

**링크 단계의 라이브러리 역할 :**

* 기능 연결: 미리 구현된 외부 함수의 구현부를 사용자의 코드와 연결해 준다.
* 결합 방식:

	* 정적 링크: .lib 파일의 코드를 실행 파일 안에 직접 포함시켜 실행 파일의 크기는 커지지만 별도의 외부 파일 없이 단독 실행이 가능하다.
	* 동적 링크: 실행 파일에는 함수의 위치 정보만 담고, (이때 정적 링크가 사용된다.) 프로그램 실행 시 .dll 파일을 참조한다. 이러한 경우 여러 프로그램이 동일한 라이브러리를 공유할 수 있어 메모리 효율이 증가한다.
	

## 6.정적 라이브러리와 동적 라이브러리 에 대하여 조사하라

	## 특히 각 라이브러리가 실행파일에 링크되는 시점을 자세히 설명하라##

**정적 라이브러리** : 컴파일된 함수들의 목적 코드를 하나의 파일로 묶어놓은 것이다.

* 링크 시점: 빌드 타임 중 링크 단계에서 실행 파일 내부에 라이브러리 코드가 직접 포함된다.

**동적 라이브러리** : 프로그램 실행 시에 로드하여 사용할 수 있도록 컴파일된 파일이다.

* 링크 시점: 런타임, 즉 프로그램이 실제로 실행되는 순간에 메모리에 호출, 적재되어 연결된다.
