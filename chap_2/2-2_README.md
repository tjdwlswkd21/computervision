##1.비주얼 스튜디오 프로젝트 속성에서 Debug 모드 Release 모드의 차이를 설명하라##

**디버그 모드** : 소프트웨어 개발 단계에서 버그를 찾고 문제를 해결하기 위해 사용하는 빌드 모드이다.

	* **주요 특징: **

		* 디버그 정보 포함: 실행 파일에 소스 코드와 연결된 디버그 정보를 포함하여 중단점을 설정하거나 변수 값을 실시간으로 확인할 수 있다.
		* 최적화 미수행: 코드의 실행 순서가 소스 코드와 일치하도록 컴파일러 최적화를 수행하지 않아 실행 속도가 상대적으로 느려진다.
		* 메모리 검사: 메모리 할당 시 오류를 감지하기 위한 추가적인 코드가 삽입된다.


##2. 비주얼 스튜디오 프로젝트 속성에서 x 86 모드 x 64 모드의 차이를 설명하라##

**릴리스 모드** : 개발을 완료한 후 사용자에게 배포하기 위해 성능을 극대화한 빌드 모드이다.

	* **주요 특징: **

		* 코드 최적화: 컴파일러가 속도와 크기를 기준으로 코드를 최적화해 불필요한 코드를 제거하거나 실행 순서를 변경하여 실행 속도를 향상시킨다.
		* 디버그 정보 제거: 실행 파일의 크기를 줄이기 위해 불필요한 디버깅용 정보를 제거하여 디버깅이 어려워진다.
		* 속도 및 크기 우선: 메모리 검사 등의 부가적인 코드를 생략하여 실제 구동 성능을 중점적으로 고려하여 실행한다.


##3. 빌드 전에 비주얼 스튜디오에서 포함 디렉토리를 설정하는 이유를 설명하라##

**포함 디렉터리 설정** : C++ 소스 코드에서 #include 문을 통해 불러오는 헤더 파일들의 위치를 컴파일러에게 알려주는 설정이다.

	* **설정 이유:**

		* 외부 라이브러리 인식: OpenCV와 같은 외부 라이브러리는 비주얼 스튜디오 기본 경로에 설치되어 있지 않으므로 헤더 파일이 어디에 있는지 경로를 직접 지정해 주어야 컴파일러가 해당 함수의 정의를 찾을 수 있게 된다.
		* 경로 간소화: 이 설정을 마치야만 소스 코드 작성 시 파일의 전체 경로가 아닌 파일 이름만으로 간결하게 불러올 수 있다.
		* 컴파일 오류 방지: 설정을 하지 않을 경우, 컴파일러는 해당 헤더 파일을 찾지 못해 C1083: Cannot open include file 에러를 발생시킨다.
	


##4. 빌드 전에 비주얼 스튜디오에서 라이브러리 디렉토리를 설정하는 이유를 설명하라##

**라이브러리 디렉터리 설정** : 컴파일된 함수의 실제 구현체가 담긴 라이브러리 파일의 위치를 링커에게 알려주는 과정이다.

##5. 빌드 전에 비주얼 스튜디오에서 링커 추가종속성을 설정하는 이유를 설명하라##

**링커 추가 종속성 설정** : 프로젝트에서 사용할 특정한 라이브러리 파일(.lib)의 이름을 링커에게 명시적으로 알려주는 과정이다.

	* **설정 이유:**

		* 파일 식별: 라이브러리 디렉터리 설정이 라이브러리들이 모여 있는 폴더를 알려주면, 추가 종속성은 그 폴더 안의 파일 중 어떤 파일을 사용할지 확정 짓는 단계이다.
		* 심볼 연결: 컴파일된 오브젝트 파일이 호출하는 함수가 어느 .lib 파일에 저장되어 있는지 연결하여 최종 실행 파일을 완성한다.
		* 빌드 오류 해결: 라이브러리 경로가 맞더라도 이 설정에서 파일명을 빠뜨리면 링커는 함수의 실체를 찾지 못해 LNK2019 또는 LNK1104 에러를 발생시킨다.


##6. DLL 를 사용하기 위하여 시스템 환경변수 Path 에 추가하는 경로는 무엇이고 왜 추가해야 하는지 설명하라##

**DLL 사용을 위한 Path 환경변수 설정 경로** : 사용할 프로그램(OpenCV)이 설치된 폴더 내의 실행 파일(바이너리) 폴더의 경로이다.

	* **Path에 추가해야 하는 이유:**

		* 런타임 위치 탐색: 동적 라이브러리(DLL)는 프로그램이 실행되는 시점에 연결되는데, 윈도우 운영체제는 실행 파일이 위치한 폴더에 해당 DLL이 없다면 시스템 환경변수 Path에 등록된 경로들을 순서대로 뒤져서 파일을 찾는다.
		* 실행 오류 방지: Path 설정이 되어 있지 않으면 프로그램을 실행할 때 "컴퓨터에 opencv_world4120.dll이(가) 없어 프로그램을 시작할 수 없습니다"라는 시스템 오류 메시지가 발생하며 실행이 중단된다.
		* 공유 및 효율성: 라이브러리 파일을 모든 프로젝트 폴더마다 복사해 넣지 않아도, Path에 한 번만 등록해 두면 시스템 어디에서든 해당 DLL을 참조하여 프로그램을 구동할 수 있다.


##7. 자신의 사진을 찍고 사진파일을 화면에 출력하는 프로그램을 작성하라##
``` cpp
#include <opencv2/opencv.hpp>   // opencv 헤더파일 포함
#include <iostream>             // c++ 헤더파일 포함
using namespace cv;             // cv 이름영역 생략
using namespace std;            // std 이름영역 생략
int main() {                    // 메인함수 선언
    cout << "Hello OpenCV " << CV_VERSION << endl;  // 문자열, cv버전 출력
    Mat img = imread("my_face.png");                // Mat 객체 img 선언, 이미지 파일 읽어오기 후 저장
    if (img.empty()) {                              // img객체에 데이터가 없으면 아래 코드 블록 실행
        cerr << "Image load failed!" << endl;       // 이미지 로드 오류 안내문자열 출력
        return -1;              // 비정상 종료
    }                           // 조건문 종료
    namedWindow("image");       // image 라는 이름의 영상을 출력할 윈도우 생성
    imshow("image", img);       // 윈도우에 영상 출력
    waitKey(0);                 // 아무 키가 입력될 때까지 현재 상태로 대기
    return 0;                   // 정상 종료
}                               // 메인함수 종료
```
